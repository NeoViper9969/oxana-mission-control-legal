Oxana Mission Control

Privacy Policy

Last updated: July 16, 2026

What this is

Oxana Mission Control is a personal automation application built and used exclusively by its developer, Jonathan Hayes, to manage his own social media accounts for one project, Sam Says. It is not a public product, is not distributed to other users, and is not offered as a service to anyone besides its developer.

What data this application accesses

Via the official APIs of the platforms the developer connects (currently YouTube and TikTok, with additional platforms such as Instagram planned), solely for accounts the developer owns and operates:

  • Account and video metadata (titles, descriptions, upload/publish status)
  • The ability to upload or schedule videos to the developer's own account(s)
  • Basic account analytics (views, engagement) where the platform's API provides it

Access is authorized once through each platform's own OAuth consent screen.

How this data is used

To power a local dashboard the developer uses to review analytics and manage publishing for his own channel, and to let that dashboard recognize newly published content so it can update itself. Data is never analyzed, aggregated, or used for any purpose beyond operating this one channel across the platforms it publishes to.

Where this data is stored

Entirely on the developer's own personal computer. Authorization tokens and cached responses are stored locally in a private, access-restricted folder. Nothing is uploaded to any server, cloud service, or third party.

Data sharing

None. This application never sells, shares, or discloses data to any third party. The only data that ever leaves the developer's machine is: requests made directly to each platform's own API, and, optionally, brief operational status messages (for example, "a video finished processing") sent to a private notification channel the developer personally controls — these contain no account credentials and no viewer or follower personal data.

Data retention and deletion

Authorization tokens remain stored locally until the developer revokes access through the relevant platform's own account settings — revocation immediately and permanently stops all API access, and this application deletes its locally cached tokens on the next run.

Children's privacy

This application is not directed at children and processes no data belonging to any end user other than its single developer-operator.
